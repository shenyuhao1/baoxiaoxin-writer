**程序规划与实现思路整理**

你想要的是一个**纯C语言**（Win32 API）的桌面小工具，核心功能是：把文本（手动输入或从文件加载）通过**模拟键盘输入**（调用操作系统API）逐字符打到目标窗口（如网页输入框），从而绕过“禁止粘贴”的限制。同时要求UI美观、操作丝滑、必须线程隔离。

下面我把整个思路从需求 → 架构 → 实现细节 → 技术选型 → 开发步骤完整梳理一遍，方便你直接照着做。

### 1. 需求拆解（核心功能 + 体验要求）

**必备功能**
- UI中提供多行文本编辑框，支持手动粘贴/输入文字
- 支持从本地文件（.txt、.md 等）一键加载文本
- 可调节“每个字符输入间隔”（毫秒级，默认50~300ms）
- 可调节“整体输入速度”（可选：慢/中/快 预设）
- 点击“开始输入”后，程序自动把文本逐字符模拟键盘发送到**当前激活的输入框**
- 支持暂停/继续/停止
- 支持输入进度实时显示（已输入多少字符/总字符）

**体验要求（丝滑 + 好看）**
- UI不能卡顿（必须主线程只负责界面，输入模拟放工作线程）
- 界面现代化：圆角、阴影、现代配色、字体清晰（支持中文）
- 操作流畅：热键启动（Ctrl+Shift+V）、拖拽调整窗口、状态栏提示
- 防止误操作：开始前弹出“请先把光标定位到目标输入框”的提示

**非功能要求**
- 纯C语言 + Win32 API（不引入C++）
- 跨Windows版本兼容（Win10/11为主）
- 体积小（<2MB）、无外部DLL依赖
- 支持中文文本输入（Unicode）

### 2. 整体架构（线程隔离是核心）

```
主线程（UI线程）
├── 创建窗口、控件（Edit、Button、Trackbar、Static等）
├── 消息循环（GetMessage + DispatchMessage）
├── 事件处理（加载文件、开始/暂停/停止按钮、设置延迟）
└── 只负责界面刷新、状态显示

工作线程（输入模拟线程）← 用 _beginthreadex 创建
├── 接收主线程传递的文本缓冲区 + 延迟参数
├── 循环遍历每个字符
├── 调用 SendInput 发送键盘事件
├── 每发一个字符 sleep(用户设置的间隔)
├── 通过事件/标志位与主线程通信（暂停、停止、进度更新）
└── 结束后通知主线程更新UI
```

**线程安全关键点**：
- 用 `CRITICAL_SECTION` 保护共享的文本缓冲和进度变量
- 用 `HANDLE` 事件对象（`CreateEvent`）实现暂停/继续/停止信号
- UI更新全部通过 `PostMessage` 或 `SendMessage` 回主线程

### 3. 核心技术选型（纯C可行方案）

| 模块         | 技术方案                          | 理由 |
|--------------|-----------------------------------|------|
| **GUI**      | Win32 API + ComCtl32 + 自定义OwnerDraw | 纯C原生，最轻量；通过视觉样式（`InitCommonControlsEx` + manifest）实现现代感 |
| **文本编辑** | 多行 `EDIT` 控件 + `EM_SETTEXTEX` | 支持Unicode，方便大文本 |
| **文件加载** | `GetOpenFileName` + `CreateFile` + `ReadFile` | 原生API |
| **模拟输入** | `SendInput`（推荐）或 `keybd_event` | SendInput 支持Unicode字符（`KEYEVENTF_UNICODE`），完美处理中文 |
| **中文输入** | `SendInput` + `KEYEVENTF_UNICODE` | 直接发 wchar_t，不依赖输入法 |
| **线程**     | `_beginthreadex` + `CRITICAL_SECTION` + Event | 比 `CreateThread` 更安全 |
| **配置保存** | 简单INI文件（`WritePrivateProfileString`） | 无需额外库 |

**UI美化技巧（纯Win32也能很好看）**：
- 用 `SetWindowTheme` + `uxtheme.dll` 开启现代视觉样式
- 自定义绘制按钮/进度条（WM_DRAWITEM）
- 窗口圆角（`CreateRoundRectRgn` + `SetWindowRgn`）
- 暗黑模式支持（Win10 1809+ 可通过 `DwmSetWindowAttribute`）
- 字体：`CreateFont` 用微软雅黑 10-12pt

### 4. 用户操作流程（丝滑体验设计）

1. 打开程序 → 看到清爽主界面
2. 在左侧大文本框粘贴或点击“从文件加载”
3. 右侧面板调节“字符间隔”（Trackbar + 编辑框同步）
4. 点击“准备输入” → 弹出提示：“请把鼠标焦点切换到网页输入框，然后点击‘开始’”
5. 用户切换到浏览器，把光标点进目标输入框
6. 点击“开始输入” → 工作线程启动，界面实时显示进度条 + “正在输入：第123/4567个字符”
7. 支持中途点击“暂停” / “停止”

额外丝滑功能（强烈建议加上）：
- 全局热键（`RegisterHotKey`）：Ctrl+Alt+V 直接开始
- 输入完成后自动提示音 + 状态栏显示“输入完成”
- 记忆上次输入间隔（INI保存）

### 5. 开发步骤建议（分阶段）

**阶段1：基础框架（1-2天）**
- 创建Win32项目，搭建主窗口 + 控件布局
- 实现文本框 + 文件打开

**阶段2：线程隔离（1天）**
- 创建工作线程框架，传递文本和参数
- 用Event实现暂停/停止

**阶段3：键盘模拟核心（最重要）**
- 写一个 `SimulateTyping(const wchar_t* text, int delay_ms)` 函数
- 处理特殊字符（回车、Tab、空格等）
- 支持Unicode（中文）

**阶段4：UI美化 + 体验打磨（2-3天）**
- 视觉样式、圆角、暗黑模式
- 进度显示、状态栏
- 热键、提示弹窗

**阶段5：打包与优化**
- 清单文件（manifest）开启视觉样式
- 编译Release x64
- 添加“关于”页面，写上版本和作者

### 6. 潜在难点 & 解决方案

- **中文输入**：必须用 `KEYEVENTF_UNICODE` + `INPUT` 结构体发 `wchar_t`，不能用VK码
- **网页反自动化**：部分网站会检测输入速度过快 → 所以必须支持可调延迟（建议默认80ms）
- **焦点问题**：程序无法自动把焦点给浏览器 → 必须用户手动点进去（这是所有模拟输入工具的通用限制）
- **管理员权限**：某些网页/程序需要以管理员运行工具才能模拟输入 → 可选添加“以管理员身份运行” manifest
